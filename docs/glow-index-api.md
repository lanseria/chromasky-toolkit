# 火烧云指数点查询 API

## 概述

根据经纬度、事件类型（日出/日落）和日期，查询该点的火烧云指数。

接口会先通过天文计算得出该地点在指定日期的真实日出/日落时间（北京时间），然后从已有数据中匹配最近的时间点，返回该点的完整评分。

## 请求

```
GET /api/glow-index
```

### 参数

| 参数 | 类型 | 必填 | 范围 | 说明 |
|------|------|------|------|------|
| `lat` | float | 是 | -90 ~ 90 | 纬度 |
| `lon` | float | 是 | -180 ~ 180 | 经度 |
| `event` | string | 是 | `sunrise` / `sunset` | 日出或日落 |
| `date` | string | 是 | `YYYY-MM-DD` | 查询日期 |

### 请求示例

```
GET /api/glow-index?lat=39.9&lon=116.4&event=sunset&date=2026-05-14
```

## 响应

### 成功响应 (200)

| 字段 | 类型 | 说明 |
|------|------|------|
| `lat` | float | 请求的纬度 |
| `lon` | float | 请求的经度 |
| `date` | string | 查询日期 |
| `event` | string | 事件类型 |
| `event_time` | string | 该点真实的日出/日落时间（北京时间，HH:MM） |
| `data_time` | string | 匹配到的数据时间点（北京时间，HH:MM） |
| `final_score` | float | 最终火烧云指数（0 ~ 1，越高越好看） |
| `score_boundary` | float | 云边界距离评分（品质因子） |
| `score_hcc` | float | 高云覆盖率评分（品质因子） |
| `score_mcc` | float | 中云覆盖率评分（品质因子） |
| `score_lcc` | float | 低云遮挡评分（惩罚因子） |
| `score_aod550` | float | 大气透明度评分（惩罚因子） |

### 响应示例

```json
{
    "lat": 39.9,
    "lon": 116.4,
    "date": "2026-05-14",
    "event": "sunset",
    "event_time": "19:23",
    "data_time": "19:00",
    "final_score": 0.7523,
    "score_boundary": 0.8125,
    "score_hcc": 0.9,
    "score_mcc": 0.7,
    "score_lcc": 1.0,
    "score_aod550": 1.0
}
```

### 错误响应

| 状态码 | 说明 | 示例 |
|--------|------|------|
| 400 | 参数格式错误 | 日期格式错误 / 该地点在该日无日出日落事件 |
| 404 | 数据不存在 | 指定日期没有下载过气象数据 |
| 422 | 参数缺失或类型不对 | 缺少必填参数 |

```json
{
    "detail": "未找到 2026-05-14 的数据，请确认该日期已有数据下载"
}
```

## 评分模型

`final_score` 由品质分和惩罚分两部分组成：

```
最终得分 = 品质分 × 惩罚分
```

**品质因子**（加权平均）：

| 因子 | 权重 | 说明 |
|------|------|------|
| `score_boundary` | 0.8 | 云边界距离，400km 最佳，超过 500km 为 0 |
| `score_hcc` | 0.1 | 高云覆盖，40%~80% 最佳 |
| `score_mcc` | 0.1 | 中云覆盖，20%~50% 最佳 |

**惩罚因子**（相乘）：

| 因子 | 说明 |
|------|------|
| `score_lcc` | 低云遮挡，≤10% 不扣分，>50% 直接归零 |
| `score_aod550` | 大气透明度，AOD < 0.3 不扣分，> 0.6 归零 |

## 数据来源

接口按优先级查找数据：

1. **预计算结果** — `outputs/calculations/{date}/glow_index_result_{HHMM}.nc`（定时任务已计算的全网格结果，直接插值，速度快）
2. **原始气象数据** — `data/processed/future/{date}/`（实时加载 GFS 云量 + CAMS 气溶胶数据并计算）

如果指定日期没有运行过数据获取流程，接口将返回 404。

## 时间匹配逻辑

1. 根据经纬度和日期计算真实的日出/日落 UTC 时间（天文算法）
2. 转换为北京时间
3. 在可用数据时间点中选取差值最小的一个

可用数据时间点随季节变化：

| 季节 | 月份 | 日出时间点 | 日落时间点 |
|------|------|-----------|-----------|
| 冬季 | 11, 12, 1, 2 | 06:00 ~ 09:00 | 17:00 ~ 20:00 |
| 夏季 | 5, 6, 7, 8 | 04:00 ~ 07:00 | 19:00 ~ 22:00 |
| 春秋季 | 3, 4, 9, 10 | 05:00 ~ 08:00 | 18:00 ~ 21:00 |
